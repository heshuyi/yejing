## 1. 产品概述

### 1.1 一句话定位

**野径**是一款面向个人徒步爱好者的原生 iOS App：在地图上记录 GPS 轨迹，沿途标记补给点与美照位置，出发前规划如何到达起终点，徒步中与 Apple Watch 同步健康数据，形成可反复查阅的「个人徒步档案」。

### 1.2 核心价值

| 用户痛点 | 野径解法 |
|----------|----------|
| 徒步后只记得里程，不记得哪里补水、哪里好拍 | 地图锚点 + 照片 + 备注，按路线归档 |
| 信息散落在相册和备忘录 | 以路线为中心统一存储 |
| 户外掏手机不便 | Watch 独立记录 + 一键标记 |
| 不知道怎么到 trailhead | 起终点交通方案 + 跳转系统地图 |
| 运动数据与路线记忆割裂 | HealthKit 与 GPS 时间轴对齐 |

### 1.3 产品边界

| 做 | 不做 |
|----|------|
| 个人路线记录与标记 | 社交广场、陌生人约伴 |
| 跳转系统地图导航 | 自建导航引擎 |
| HealthKit / Watch 深度整合 | Android 版 |
| GPX 导入导出 | 商业路线售卖 |

## 2. 目标用户

### 2.1 核心用户：周末徒步记录者

- 每月 1–4 次周边徒步
- 使用 iPhone + Apple Watch
- 需要「可再次参考」的补给点、美照机位档案
- 关注：里程爬升、心率、交通、拍照

### 2.2 次要用户

- **摄影向徒步者**：重视美照点位、朝向、黄金时段
- **分段徒步者**：多日路线按 Segment 管理

## 5. 功能需求详述

### 5.1 首页 — `screens_01-home`

**用户故事**：作为徒步者，我希望打开 App 就能继续上次未完成的记录，或快速开始新路线。

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 进行中卡片 | 展示路线名、里程、爬升、实时心率 | P0 |
| 继续记录 | 恢复后台 GPS 会话 | P0 |
| 添加标记 | 跳转记录页并打开标记 Sheet | P0 |
| 最近路线 | 列表展示，点击进入详情 | P0 |
| Watch 状态 Pill | 已连接 / 未连接 / 同步中 | P0 |
| 快捷入口 | 标记、到达、Watch、规划 | P0 |
| 底部 Tab | 首页 / 路线 / 地图 / 我的 | P0 |
| 空状态引导 | 首次使用创建第一条路线 | P1 |

**验收标准：**
- [ ] 有进行中路线时，卡片 1 秒内展示最新里程

### 5.2 GPS 记录 — `screens_02-recording`

**用户故事**：作为徒步者，我希望在行走中实时看到轨迹和关键指标，并能快速标记当前位置。

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 后台 GPS 记录 | `CLLocationManager` 户外精度 | P0 |
| 实时地图轨迹 | MapKit polyline + 当前位置 | P0 |
| 核心指标 | 距离、爬升、配速、时长 | P0 |
| 暂停/继续 | 暂停不计入距离 | P0 |
| 结束保存 | 确认后写入路线 + HealthKit | P0 |
| Watch 心率条 | 实时心率 + 区间文案 | P0 |
| 标记 Bottom Sheet | 补给 / 美照 / 休息 三选一 | P0 |
| GPS 弱信号提示 | Banner + 虚线轨迹 | P1 |
| 低电量模式 | <15% 提示降采样 | P1 |

**指标计算：**
- 爬升：海拔上升累计，±5m 防抖
- 配速：500m 滑动窗口均值
- 距离：5m 最小间隔抽稀后累加

**验收标准：**
- [ ] 点击「+」弹出标记类型选择，默认继承上次类型
- [ ] 暂停后指标冻结，继续后恢复
- [ ] 结束保存后路线状态变为「已完成」
- [ ] 4 小时连续记录 App 不被系统杀死（iOS 17+ Background Session）

### 6.1 MVP 类型

### 6.2 v1.1 扩展

观景点、岔路/路标、露营点、其他

### 6.3 标记数据规则

- 坐标：WGS84，精度随 GPS 记录
- `distanceFromStart`：沿轨迹累计距离自动计算
- 照片：本地存储，最多 9 张/标记
- 时间戳：创建时间 + 最后编辑时间

## 8. 用户旅程

### 8.1 行前

```
打开野径 → 路线库选「雾凇岭环线」→ 查看标记地图
→ 如何到达 → 导航至停车场 → 到达后「开始记录」
```

### 8.2 行中

```
iPhone 后台记录 + Watch 看心率
→ 观景台：Watch 点「美照」或 iPhone「+」选美照
→ 驿站：标记补给并拍照
→ 休息：暂停
→ 终点：结束 → 保存 Workout + 路线档案
```

### 8.3 行后

```
路线详情 → 查看轨迹与心率对照
→ 给美照点补照片、朝向备注
→ 下次徒步前再次参考
```

## 9. 数据模型

```
User（本地，无账号）
 └── Route
      ├── id, name, status: draft | active | completed
      ├── startCoordinate, endCoordinate
      ├── isLoop: Bool
      ├── segments[]（可选）
      ├── trackPoints[]: { timestamp, lat, lng, altitude, speed }
      ├── markers[]: { id, type, name, note, coordinate,
      │                 distanceFromStart, photos[], facing?, bestTime? }
      └── healthSession: { workoutUUID, heartRateSamples[],
                           calories, steps, spo2, elevationGain }
```

**存储**：SwiftData / Core Data 本地；照片 FileManager；iCloud 可选同步。

## 12. 验收标准（MVP 整体验收）

### 12.1 核心闭环

### 12.2 标记

- [ ] 补给点、美照位置可添加、编辑、删除
- [ ] 地图与列表数据一致
- [ ] 美照点包含朝向、推荐时段

### 12.3 Watch
